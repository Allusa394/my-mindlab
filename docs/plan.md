## ШАГ 0 — Контекст методологии

**Цель:** Дать AI знание о конкретной методологии тренинга, чтобы блоки не были шаблонными.

**Что делаем:**
- Получаем материал из ТГ-группы (тексты, концепции, термины методологии)
- Вносим в `research.md` как раздел "Контекст методологии"
- Формируем системный промпт для AI на основе этого контекста

**Результат:** AI знает язык и концепции методологии → блок НЕЙРОКОНТУР и СВЯЗЬ С СОБОЙ становятся точными, а не общими.

**Файлы:** `research.md` (раздел "Контекст методологии"), `prompts.md` (системный промпт)

## ШАГ 1 — Быстрый захват

**Цель:** Убрать барьеры в основном сценарии: "слушаю тренинг → появилась мысль → зафиксировал за 10 секунд".

**Проблема сейчас:** 3 действия перед вводом (тема + сфера + текст). Когнитивный шум во время потока.

**Что делаем:**
- Режим "Быстрая мысль" — одно поле textarea, одна кнопка "Анализировать"
- Тема и сфера заполняются автоматически (по ключевым словам) или после анализа
- Мобильная оптимизация: крупные тапабельные зоны, минимум прокрутки, безопасные зоны iOS

**Результат:** Пользователь фиксирует мысль за 1 действие.

**Файлы:** `index.html` (новый режим быстрого захвата)

## ШАГ 2 — Редактирование блоков

**Цель:** Дать пользователю контроль над результатом AI.

**Проблема сейчас:** AI сгенерировал → исправить нельзя → доверие снижается.

**Что делаем:**
- Каждый из 6 блоков становится редактируемым (click-to-edit или inline textarea)
- Кнопка "Сохранить правки" для каждого блока
- Иконка "Перегенерировать этот блок" (после подключения реального AI)

**Результат:** Пользователь доверяет системе — он владеет результатом.

**Файлы:** `index.html` (редактируемые блоки)

## ШАГ 3 — Реальный Claude API

**Цель:** Заменить симулированные шаблонные ответы на реальный AI с контекстом.

**Проблема сейчас:** После 3–4 анализов пользователь чувствует повторение и перестаёт доверять системе.

**Что делаем:**
- Подключаем Claude Haiku API (оптимален по цене/скорости)
- Промпт содержит: контекст методологии + личный профиль пользователя + инструкции 6-блочной системы
- Backend: минимальный Node.js/Edge Function для хранения API-ключа (не в браузере)
- Обработка ошибок и состояний загрузки

**Стоимость:** ~$0.001–0.003 за один анализ (Haiku)

**Результат:** AI реально понимает контекст тренинга → блоки точные и неожиданные.

**Файлы:** `api/analyze.js` (backend), `index.html` (реальные вызовы API)

## ШАГ 4 — Spaced Repetition

**Цель:** Якорные фразы возвращаются к пользователю через 1/3/7/21 день → нейронное закрепление.

**Проблема сейчас:** Блок ЯКОРЬ генерируется, но никогда не возвращается. Система не работает без повторения.

**Что делаем:**
- Планировщик напоминаний по каждой якорной фразе
- Интерфейс "Сегодня повторить" в библиотеке
- (В Telegram-версии) бот присылает якорную фразу в нужный день

**График повторений:**
- День 1: якорная фраза сразу после анализа
- День 3: первое напоминание
- День 7: второе напоминание
- День 21: финальное закрепление

**Результат:** Ключевые инсайты остаются в голове, а не в архиве.

**Файлы:** `index.html` (планировщик + UI напоминаний)

## ШАГ 5 — Telegram Mini App / Синхронизация

**Цель:** Работа на телефоне прямо в Telegram, синхронизация между устройствами.

**Проблема сейчас:** localStorage — данные только на одном устройстве. Основной сценарий (тренинг в ТГ) требует телефон + компьютер.

**Что делаем:**
- Telegram Mini App: текущий index.html адаптируется как TMA
- Backend с пользовательскими аккаунтами через Telegram auth
- База данных заметок в облаке (Supabase или аналог)
- Голосовой ввод через Telegram (Voice → Whisper API → текст → анализ)

**Результат:** Записал на телефоне → открыл на компьютере → всё на месте.

**Файлы:** Новая архитектура с backend

## КРИТИЧЕСКИЕ МЕТРИКИ УСПЕХА

- **Шаг 1:** Время от открытия до начала ввода < 5 секунд
- **Шаг 3:** Пользователь говорит "это про мой тренинг" после первого анализа
- **Шаг 4:** Пользователь возвращается на день 3 и помнит якорную фразу
- **Шаг 5:** Сценарий "записал на телефоне → открыл на компьютере" работает без трений
